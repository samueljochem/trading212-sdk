# Trading 212 API - Installation & Setup Guide

## Quick Setup

### 1. Get Your API Credentials

1. Open the Trading 212 mobile app or web app
2. Navigate to **Settings → API**
3. Generate a new **API Key** and **API Secret**
4. Keep these credentials safe (never commit to version control)

### 2. Install the Package

```bash
npm install trading-212-api
```

### 3. Use in Your Code

```typescript
import { Trading212Client } from 'trading-212-api';

const client = new Trading212Client({
  apiKey: 'your-api-key',
  apiSecret: 'your-api-secret',
  environment: 'demo' // or 'live'
});

// Get account summary
const account = await client.account.getSummary();
console.log(account.totalValue);
```

## API Environments

### Demo (Paper Trading)
- Perfect for testing and development
- No real money involved
- Base URL: `https://demo.trading212.com`

```typescript
const client = new Trading212Client({
  apiKey: 'your-key',
  apiSecret: 'your-secret',
  environment: 'demo'
});
```

### Live (Real Money)
- For production trading
- Uses real funds
- Base URL: `https://live.trading212.com`

```typescript
const client = new Trading212Client({
  apiKey: 'your-key',
  apiSecret: 'your-secret',
  environment: 'live'
});
```

## Configuration Options

```typescript
interface Trading212ClientConfig {
  // Required
  apiKey: string;              // Your API Key
  apiSecret: string;           // Your API Secret

  // Optional
  environment?: 'live' | 'demo';  // Default: 'demo'
  baseUrl?: string;             // Override API endpoint
  timeout?: number;             // Request timeout in ms (default: 30000)
}
```

## Error Handling

```typescript
try {
  const order = await client.orders.placeMarket({
    ticker: 'AAPL_US_EQ',
    quantity: 10
  });
} catch (error) {
  const message = error instanceof Error ? error.message : String(error);
  
  if (message.includes('401')) {
    console.log('Authentication failed - check credentials');
  } else if (message.includes('429')) {
    console.log('Rate limited - wait before retrying');
  } else if (message.includes('403')) {
    console.log('Permission denied - check API scopes');
  } else {
    console.log('Error:', message);
  }
}
```

## Running Examples

Set up environment variables first:

```bash
# On Linux/Mac
export TRADING212_API_KEY="your-key"
export TRADING212_API_SECRET="your-secret"

# On Windows (PowerShell)
$env:TRADING212_API_KEY = "your-key"
$env:TRADING212_API_SECRET = "your-secret"
```

Then run examples:

```bash
npx ts-node examples/01-account-overview.ts
npx ts-node examples/02-positions.ts
npx ts-node examples/03-orders.ts
```

## TypeScript Support

The library is fully typed with TypeScript. You get full IntelliSense and type checking:

```typescript
// Full type support
const positions: Position[] = await client.positions.getAll();

// Type checking for requests
await client.orders.placeMarket({
  ticker: 'AAPL_US_EQ',
  quantity: 10,
  extendedHours: false
});

// All return types are properly typed
const order: Order = await client.orders.getById(123);
```

## Rate Limiting

Access rate limit information from responses:

```typescript
// All methods return rate limit info
const response = await client.account.getSummary();

if (response.rateLimitRemaining! < 5) {
  console.log('Running low on requests');
}

const resetTime = new Date(response.rateLimitReset! * 1000);
console.log('Rate limit resets at:', resetTime);
```

## Troubleshooting

### Module not found errors
```bash
# Make sure to build first
npm run build
```

### Type errors in TypeScript
- Ensure `skipLibCheck: true` is set in your tsconfig.json
- Import types from the package: `import { Order, Position } from 'trading-212-api'`

### API errors
- Check your credentials are correct
- Verify you're using the right environment (demo vs live)
- Check rate limits haven't been exceeded
- Review the API docs at https://t212.dev/
